# Doodly 영상 촬영 및 발표 가이드

README의 영상 링크는 아래 순서로 촬영한 뒤 YouTube 공개 또는 일부 공개 영상 URL로 교체합니다.

## 1. 2분 요약 영상 구성

### 0:00-0:15 앱 소개

화면: Doodly 시작 화면 → 작성 화면

대본:

> Doodly는 한두 문장의 짧은 일기와 오늘의 감정을 입력하면 AI가 따뜻한 손그림으로 바꿔 주는 비주얼 다이어리입니다. 긴 일기의 부담을 줄이고, 다시 보고 싶은 하루를 만드는 것이 목표입니다.

### 0:15-0:55 작성과 AI 그림 생성

화면: 감정 선택 → 일기 입력 → AI로 그림 그리기 → 생성 애니메이션 → 결과

대본:

> 날짜와 감정을 선택하고 짧은 일기를 입력합니다. AI로 그림 그리기를 누르면 Gemini가 내용을 이미지 프롬프트로 정리하고 손그림 이미지를 생성합니다. 생성 중에는 애니메이션을 표시하고, 실패 시에도 로컬 폴백 이미지를 만들어 작성 흐름이 중단되지 않습니다.

### 0:55-1:25 갤러리와 감정 캘린더

화면: 저장 → 갤러리 → 캘린더 → 요즘 내 감정 보기

대본:

> 저장한 기록은 갤러리에서 그리드나 리스트로 볼 수 있습니다. 감정 캘린더에는 날짜별 감정과 이미지가 표시되고, 최근 7일 기록을 바탕으로 AI가 짧은 감정 메시지를 제공합니다. 미래 날짜에는 기록할 수 없고, 날짜별 한 편만 저장됩니다.

### 1:25-1:45 공유와 알림

화면: 상세 화면 공유 스위치 → 공유 시트 → 설정 알림

대본:

> 상세 화면에서는 일기 글 포함 여부를 선택해 그림만 또는 그림과 글을 공유할 수 있습니다. 설정에서는 매일 알림 시각을 정할 수 있고, 알림을 누르면 딥링크로 작성 화면이 열립니다.

### 1:45-2:00 기술 요약

화면: 아키텍처 이미지 또는 Android Studio 프로젝트 구조

대본:

> Kotlin과 Jetpack Compose로 UI를 구현했고, MVVM과 Repository 구조, Room Flow, Retrofit, Gemini, AlarmManager, FileProvider를 사용했습니다. Doodly는 짧은 입력을 시각적 회고 경험으로 바꾸는 앱입니다.

## 2. 10분 상세 발표 구성

### 0:00-1:00 문제 정의와 기획 의도

- 긴 일기 작성의 부담
- 텍스트 기록을 다시 열어볼 동기 부족
- 짧은 입력과 AI 시각화로 해결

### 1:00-2:00 사용자 흐름과 UI 콘셉트

- 스플래시 후 작성 화면 우선
- 파스텔, 둥근 카드, 충분한 여백
- 토스식 정보 구조와 감성적인 다이어리 디자인 결합

### 2:00-4:00 핵심 기능 1: 작성과 Room 저장

- `WriteUiState`와 `WriteViewModel`
- 날짜별 unique index
- 같은 날짜 재진입 시 기존 기록 수정
- 미래 날짜 이중 차단
- MVVM, Repository, DAO 흐름 설명

### 4:00-6:00 핵심 기능 2: AI 이미지 생성

- Gemini SDK로 JSON 이미지 프롬프트 생성
- Retrofit으로 이미지 모델 호출
- Base64 디코딩과 내부 파일 저장
- Coil 표시
- 키 누락/API 실패 폴백
- 생성 애니메이션과 결과 등장

### 6:00-7:30 핵심 기능 3: 감정 캘린더

- Room 월별 Flow
- Custom Layout 7열 캘린더
- 감정 아이콘과 썸네일
- 최근 7일 데이터 집계
- Gemini AI 한마디와 로컬 메시지 폴백

### 7:30-8:20 공유, 알림, 딥링크

- FileProvider와 ACTION_SEND
- 글 포함 스위치
- AlarmManager와 BroadcastReceiver
- `doodly://write`, `doodly://diary/{id}`

### 8:20-9:20 트러블 슈팅

1. 이미지 API 응답 형식 불일치
2. 캘린더 → 작성 → 뒤로가기 백스택 문제
3. 날짜 중복과 캘린더 매핑 문제
4. 미래 날짜 기록 차단
5. API 키 보안

### 9:20-10:00 후기와 발전 방향

- Compose 상태 관리와 Room Flow 학습
- AI API는 실패 흐름까지 설계해야 함
- Firebase 동기화, 멀티모달 감정 추천, 통계, 보안 기능 확장 예정
