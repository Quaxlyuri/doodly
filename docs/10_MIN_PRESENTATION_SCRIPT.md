## 0:00-0:45 인사와 프로젝트 소개

**화면**

- Doodly 앱 아이콘
- 스플래시 화면

**발표 대본**

안녕하세요. 모바일 프로그래밍 3193분반 권태훈입니다.  
제가 개발한 프로젝트는 AI 비주얼 다이어리 앱, Doodly입니다.

Doodly는 사용자가 오늘 있었던 일을 한두 문장으로 입력하고 감정을 선택하면, AI가 그 내용을 따뜻한 손그림 이미지로 만들어 주는 Android 일기 앱입니다.

기존 일기 앱은 긴 글을 작성해야 한다는 부담이 있고, 글만 쌓이기 때문에 시간이 지난 뒤 다시 찾아보는 재미가 부족하다고 생각했습니다. 그래서 작성 부담은 줄이면서도 다시 보고 싶은 기록을 만드는 것을 프로젝트의 핵심 목표로 정했습니다.

## 0:45-1:30 기획 의도와 핵심 사용자 흐름

**화면**

- 작성 화면 전체
- 날짜, 감정 칩, 일기 입력칸을 순서대로 가리키기

**발표 대본**

Doodly의 주요 대상은 긴 일기를 쓰는 것은 부담스럽지만 하루를 간단하게 남기고 싶은 사용자입니다.

사용 흐름은 단순합니다. 앱을 실행하면 짧은 스플래시 화면 뒤에 바로 작성 화면이 열립니다. 사용자는 날짜를 확인하고 오늘의 감정을 선택한 다음, 한두 문장의 일기를 작성합니다. 이후 AI로 그림 그리기 버튼을 누르면 그림이 생성되고, 결과를 확인한 뒤 저장할 수 있습니다.

저장된 기록은 갤러리와 감정 캘린더에서 다시 확인할 수 있고, 상세 화면에서는 그림만 공유하거나 일기 글을 함께 공유할 수 있습니다.

## 1:30-2:20 UI/UX 디자인

**화면**

- 작성 화면
- 하단 내비게이션
- 감정 아이콘
- 캘린더 화면

**발표 대본**

디자인은 정보 구조가 명확한 토스 스타일을 기반으로 하고, 감성적인 다이어리 앱에 어울리도록 파스텔 색상과 큰 둥근 모서리를 적용했습니다.

화면 좌우에는 충분한 여백을 두고, 중요한 버튼은 크고 명확하게 배치했습니다. 이모지 문자를 그대로 사용하지 않고 행복, 설렘, 평온, 피곤, 슬픔, 화남을 나타내는 일관된 벡터 아이콘을 제작해 사용했습니다.

또한 “한두 문장이면 충분해요”, “짠! 오늘의 그림이 완성됐어요”와 같은 짧고 친근한 문구를 사용했습니다. 앱을 실행한 사용자가 가장 먼저 해야 하는 행동은 일기 작성이기 때문에 스플래시 화면 다음에 작성 화면이 바로 나오도록 구성했습니다.

## 2:20-3:30 시스템 아키텍처와 데이터 저장

**화면**

- Android Studio 프로젝트 구조
- `DiaryEntry.kt`
- `DiaryDao.kt`
- `WriteViewModel.kt`

**발표 대본**

앱 아키텍처는 강의에서 배운 MVVM과 Repository 패턴을 사용했습니다.

전체 흐름은 Compose UI, ViewModel, Repository, DAO, RoomDatabase 순서입니다. Compose 화면은 ViewModel의 Flow를 collectAsState로 구독합니다. ViewModel은 viewModelScope에서 비동기 작업을 실행하고, Repository가 Room과 AI 데이터 소스를 감싸도록 구성했습니다.

Room 데이터베이스의 DiaryEntry에는 날짜, 일기 내용, 감정, 이미지 경로, 태그와 생성 시각이 저장됩니다. 날짜에는 unique index를 적용해서 하루에 한 편만 저장되도록 했습니다.

같은 날짜를 다시 선택하면 새 데이터를 추가하지 않고 findByDate로 기존 기록을 불러옵니다. 이후 editingId가 있으면 update를 실행하고, 기존 기록이 없을 때만 insert를 실행합니다.

이 정책을 통해 같은 날짜의 감정이나 이미지가 캘린더에서 중복되는 문제를 방지했습니다. 태그 목록은 TypeConverter를 이용해 JSON 문자열로 변환해 Room에 저장했습니다.

## 3:30-5:00 핵심 기능 1: AI 그림 생성

**화면**

- 일기 내용 입력
- AI로 그림 그리기 버튼 누르기
- 생성 애니메이션
- 완성 이미지
- 가능하면 `AiRepository.kt` 코드

**발표 대본**

첫 번째 핵심 기능은 일기 내용을 AI 그림으로 바꾸는 기능입니다.

사용자가 버튼을 누르면 WriteViewModel이 AiRepository의 generateDiaryImage를 호출합니다. 먼저 Gemini 텍스트 모델에 일기 내용과 감정을 전달해 영어 이미지 생성 프롬프트를 만듭니다. 이때 응답 형식을 JSON으로 제한하고, image_prompt와 mood_keyword를 JSONObject로 파싱합니다.

그다음 Retrofit으로 Gemini 이미지 모델의 generateContent API를 호출합니다. 응답의 inlineData에 들어 있는 Base64 이미지 데이터를 디코딩하고, 앱 내부 저장소의 files/images 폴더에 UUID 파일명의 PNG로 저장합니다. Room에는 이미지 자체가 아니라 로컬 파일 경로를 저장하고, 화면에서는 Coil로 이미지를 불러옵니다.

AI 생성에는 시간이 필요하기 때문에 사용자가 앱이 멈췄다고 느끼지 않도록 회전과 크기 변화가 있는 파스텔 애니메이션을 추가했습니다. 생성이 끝나면 이미지가 페이드와 확대 효과로 등장하고, “짠! 오늘의 그림이 완성됐어요”라는 문구를 표시합니다.

또한 API 키가 없거나 네트워크 요청이 실패해도 앱이 종료되지 않도록 처리했습니다. 이런 경우에는 선택한 감정 색상을 기반으로 로컬 그라데이션 이미지를 만들어 저장 흐름을 계속 진행합니다.

API 키는 소스 코드에 직접 작성하지 않고 secrets.properties와 Secrets Gradle Plugin을 사용해 BuildConfig.AI_KEY로만 접근하도록 했습니다.

## 5:00-6:05 핵심 기능 2: 갤러리와 상세 공유

**화면**

- 생성 결과 저장
- 갤러리 화면
- 그리드/리스트 토글
- 기록 카드 선택
- 상세 화면 공유 옵션
- 공유 시트

**발표 대본**

두 번째 핵심 기능은 저장된 하루를 다시 보고 공유하는 기능입니다.

갤러리는 Room DAO의 전체 기록 Flow를 구독합니다. 데이터가 추가되거나 삭제되면 별도의 새로고침 없이 Compose 화면이 자동으로 다시 그려집니다.

사용자는 우측 상단 버튼으로 그리드 보기와 리스트 보기를 전환할 수 있습니다. 그리드에서는 AI 그림을 중심으로 볼 수 있고, 리스트에서는 날짜와 일기 내용까지 빠르게 확인할 수 있습니다.

기록을 누르면 상세 화면으로 이동합니다. 상세 화면에는 생성된 그림, 감정, 날짜와 일기 내용이 표시됩니다.

공유할 때는 일기 글을 함께 포함할지 스위치로 선택할 수 있습니다. ShareHelper는 이미지에 Doodly 워터마크를 합성하고 임시 파일로 저장합니다. FileProvider를 이용해 안전한 content URI를 만든 뒤 ACTION_SEND Intent로 Android 공유 시트를 호출합니다.

스위치가 켜져 있으면 그림의 워터마크와 공유 메시지에 일기 내용이 포함되고, 꺼져 있으면 그림만 공유됩니다.

## 6:05-7:15 핵심 기능 3: 감정 캘린더와 AI 한마디

**화면**

- 감정 캘린더
- 날짜별 썸네일과 감정 아이콘
- “요즘 내 감정 보기” 카드
- 이전 달 이동
- 미래 날짜가 비활성화된 모습

**발표 대본**

세 번째 핵심 기능은 감정 캘린더입니다.

CalendarViewModel은 현재 선택된 월을 StateFlow로 관리합니다. 월이 바뀌면 flatMapLatest를 이용해 해당 월의 Room 쿼리를 새로 구독합니다.

캘린더는 Compose Custom Layout으로 7열 구조를 직접 구현했습니다. 기록이 있는 날짜에는 생성된 이미지가 연하게 배경으로 표시되고, 우측 아래에는 해당 날짜의 감정 아이콘이 나타납니다. 기록이 없는 날짜를 누르면 그 날짜가 선택된 작성 화면으로 이동합니다.

미래 시점에는 감정을 기록할 수 없도록 현재보다 뒤의 날짜 셀은 투명하게 표시하고 클릭을 비활성화했습니다. 다음 달 이동도 현재 월까지만 가능하며, ViewModel에서도 날짜를 오늘 이하로 다시 보정해 잘못된 인자가 들어오는 경우까지 방어했습니다.

상단의 “요즘 내 감정 보기” 카드는 최근 7일 동안 기록된 감정의 변화를 보여 줍니다. 기록이 두 건 이상이면 날짜, 감정, 짧은 내용을 Gemini에 전달해 사용자를 판단하거나 진단하지 않는 한 문장의 메시지를 생성합니다. AI 요청이 실패하면 로컬 규칙 기반 문구를 표시하므로 화면이 비어 있지 않습니다.

## 7:15-8:00 알림과 딥링크

**화면**

- 설정 화면
- 알림 스위치
- TimePicker
- 가능하면 알림 클릭 후 작성 화면 진입

**발표 대본**

꾸준한 기록을 돕기 위해 매일 일기 알림 기능도 구현했습니다.

설정 화면에서 알림을 켜고 Material 3 TimePicker로 시간을 지정하면 AlarmManager가 다음 알림을 예약합니다. Android 13 이상에서는 accompanist-permissions를 이용해 알림 권한을 요청합니다.

알림은 BroadcastReceiver에서 표시되고, 알림을 보낸 뒤 다음 날 같은 시간의 알림을 다시 예약합니다. 알림을 누르면 doodly://write 딥링크를 통해 작성 화면이 열립니다.

상세 기록도 doodly://diary/기록아이디 형식의 딥링크를 지원합니다. PendingIntent에는 Android 12 이상에서 필요한 FLAG_IMMUTABLE을 적용했습니다.

## 8:00-9:15 트러블 슈팅

**화면**

- 생성 이미지
- 캘린더에서 날짜 선택 후 뒤로가기
- 코드의 이미지 DTO 또는 Navigation 부분

**발표 대본**

개발 과정에서 가장 크게 해결한 문제는 세 가지입니다.

첫 번째는 이미지 생성 API 문제입니다. 초기에는 이미지 모델의 요청과 응답 형식이 맞지 않아 실제 AI 그림 대신 폴백 이미지만 만들어졌습니다. 모델 엔드포인트를 generateContent 형식으로 수정하고, 응답 모달리티에 TEXT와 IMAGE를 지정했습니다. 이후 후보 응답의 part에서 inlineData.data를 찾아 Base64로 디코딩하도록 DTO를 수정해 실제 이미지 생성을 해결했습니다.

두 번째는 캘린더에서 작성 화면으로 이동한 뒤 뒤로가기가 동작하지 않는 문제였습니다. 하단 탭 이동 과정에서 백스택이 정리되면서 캘린더가 사라지는 경우가 원인이었습니다. 날짜 인자로 진입한 작성 화면에는 BackHandler를 등록하고, 캘린더가 백스택에 있으면 popBackStack을 사용하고, 없으면 현재 작성 화면을 제거하면서 캘린더를 다시 열도록 처리했습니다.

세 번째는 날짜별 데이터 중복 문제입니다. 하루에 여러 일기가 생기면 캘린더에서 어떤 감정과 이미지를 표시해야 하는지 불명확했습니다. 날짜 unique index와 기존 기록 update 방식을 적용해 날짜마다 한 편만 유지하도록 해결했습니다.

이외에도 DatePicker, 캘린더 UI, ViewModel에서 미래 날짜를 여러 단계로 차단했고, API 키는 Git에서 제외해 보안을 강화했습니다.

## 9:15-10:00 결과와 향후 발전 방향

**화면**

- 갤러리, 캘린더, 상세 화면을 빠르게 다시 보여 주기
- 마지막에 Doodly 스플래시 또는 앱 아이콘

**발표 대본**

이 프로젝트를 통해 Compose의 상태 중심 UI, Room Flow를 통한 자동 화면 갱신, Retrofit 네트워크 통신, AI 응답 처리, Navigation 백스택, AlarmManager와 FileProvider를 실제 앱 흐름 안에서 연결하는 방법을 배웠습니다.

특히 생성형 AI 기능은 정상적인 응답만 구현하는 것이 아니라 API 키 누락, 네트워크 오류, 응답 형식 변경과 같은 실패 상황까지 설계해야 안정적인 앱이 된다는 점을 배웠습니다.

현재 Doodly는 일기 작성, AI 그림 생성, Room 저장, 갤러리, 감정 캘린더, AI 감정 메시지, 공유, 알림과 딥링크 기능을 제공합니다. Firebase 백업은 Repository 인터페이스와 선택 구현을 분리해 코어 앱이 설정 파일 없이도 빌드되도록 구성했습니다.

앞으로는 사진을 활용한 멀티모달 감정 추천, 주간과 월간 감정 통계, AI 그림 스타일 선택, 기기 간 클라우드 동기화, 생체 인증과 데이터 암호화 기능을 추가하고 싶습니다.

이상으로 짧은 하루를 다시 보고 싶은 그림으로 만드는 비주얼 다이어리 Doodly 발표를 마치겠습니다. 감사합니다.

## 촬영 메모

- AI 생성 대기 시간이 길면 생성 애니메이션은 보여 주되 전체 대기 구간은 편집으로 줄입니다.
- Android Studio 코드는 글자가 읽히도록 확대합니다.
- 공유 시트의 개인 연락처와 앱 계정은 블러 처리합니다.
- 실제 발표 속도가 빠르면 각 기능 화면에서 2~3초씩 멈춥니다.
- 영상 마지막 화면에 GitHub 주소 `https://github.com/Quaxlyuri/doodly`를 표시합니다.
